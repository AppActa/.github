## Pull Request Template

```markdown
## Descrição

> Descreva brevemente o que foi alterado

## Tipo de alteração

* [ ] feat (nova funcionalidade)
* [ ] fix (correção de bug)
* [ ] docs (documentação)
* [ ] refactor (refatoração)

## Escopo

> Exemplo: login, dashboard, usuário, api, banco

## Checklist

* [ ] Código revisado
* [ ] Testes executados
* [ ] Sem conflitos de merge
* [ ] Documentação atualizada (se necessário)

## Tarefa do Jira

> Colocar link da story do Jira
```

---

## Commit Conventions

Tags principais:
```text
- feat       : nova funcionalidade
- fix        : correção de bug
- docs       : alteração de documentação
- refactor   : alteração de estrutura
```

Exemplos de escopos:
```text
- feat/login       : adiciona tela de login
- fix/api          : cria endpoint de cadastro
- docs/readme      : atualiza instruções de instalação
- refactor/usuario : reorganiza camada de serviço
```
